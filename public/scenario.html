[ SCENARIO_OVERVIEW ]

Scenario Overview

Overview // Framework

One framework, every scenario

A scenario is the structured container for planning and rehearsing a change before it reaches production. Every scenario is built from the same public building blocks: an inject-driven flow, a timeline, and an evidence trail, so a plan is always concrete and reviewable without exposing any internal tooling on this page.

The framework connects directly to the wider MSH OPS ecosystem: Scenario Engine for live execution, Mission Engine for guided delivery, Doctrine OS for approved guidance, the RGB+Gold interface system for visual consistency, and the accessibility and governance layers that keep the experience stable as it evolves.

[ SCENARIO_FLOW ]

Scenario Flow

Flow // Injects

Inject-driven, one step at a time

A scenario progresses through a sequence of injects - discrete, well-defined events that move the plan forward one step at a time. Each inject has a clear trigger and a clear outcome, so a scenario stays predictable even as it grows more complex.

Injects are organized along the same timeline-based structure used across the rest of the ecosystem, so a scenario's shape is always easy to follow at a glance - what has happened, what is happening now, and what comes next.

Injects reference the relevant Doctrine OS guidance behind the scenes, so every step stays consistent with approved practice. This page only ever shows the public-safe summary of that guidance - never the underlying doctrine content itself.

[ SCENARIO_TIMELINE ]

Scenario Timeline

Timeline // Motion-Safe

A running record, presented calmly

The scenario timeline lays out every inject in order inside a HUD-framed panel, using the same scroll-in reveal used across the rest of the site. Motion is always subtle and fully respects prefers-reduced-motion - anyone with that preference set sees the same information with no animation at all.

The layout uses the same RGB+Gold structure and HUD framing rules as the rest of the ecosystem, so timeline views remain visually consistent without adding extra motion or competing overlays.

[ SCENARIO_EVIDENCE ]

Scenario Evidence

Evidence // Audit Trail

An auditable trail of what happened

Every scenario keeps a single, chronological evidence log: inject transitions, operator actions, and agent responses all land in one place. That log is what makes a scenario auditable after the fact - nothing here is inferred or reconstructed after the fact.

The public summary above never includes engagement-specific evidence. To see a live evidence trail in action, open the Scenario Engine.
