[ MISSION_OVERVIEW ]

Mission Overview

Overview // Framework

One framework, every engagement

A mission is the structured container for an engagement - from initial briefing through delivery and debrief. Every mission is built from the same four public building blocks: objectives, a timeline, live actions, and an evidence trail, so progress is always visible and auditable without exposing any internal tooling on this page.

The framework connects directly to the wider MSH OPS ecosystem: Mission Engine for live execution, Scenario Engine for planning, Doctrine OS for approved guidance, the RGB+Gold interface system for visual consistency, and the accessibility and governance layers that keep the experience stable as it evolves.

[ MISSION_OBJECTIVES ]

Mission Objectives

Objectives // Flow

Objective-driven, checkpoint by checkpoint

Every mission is broken into a small set of concrete objectives. Each objective moves through the same checkpoint progression - planned, in progress, and complete - so it's always clear what's left before a mission can close out.

Objectives reference the relevant Doctrine OS guidance behind the scenes, so recommendations stay consistent with approved practice. This page only ever shows the public-safe summary of that guidance - never the underlying doctrine content itself.

[ MISSION_TIMELINE ]

Mission Timeline

Timeline // Motion-Safe

A running record, presented calmly

The mission timeline lays out every checkpoint in order inside a HUD-framed panel, using the same scroll-in reveal used across the rest of the site. Motion is always subtle and fully respects prefers-reduced-motion - anyone with that preference set sees the same information with no animation at all.

The layout uses the same RGB+Gold structure and HUD framing rules as the rest of the ecosystem, so checkpoint views remain visually consistent without adding extra motion or competing overlays.

[ MISSION_EVIDENCE ]

Mission Evidence

Evidence // Audit Trail

An auditable trail of what happened

Every mission keeps a single, chronological evidence log: state changes, operator actions, and agent responses all land in one place. That log is what makes a mission auditable after the fact - nothing here is inferred or reconstructed after delivery.

The public summary above never includes engagement-specific evidence. To see a live evidence trail in action, open the Mission Engine.
